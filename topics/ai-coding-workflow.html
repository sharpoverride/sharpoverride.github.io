What the AI coding tool actually does.

GitHub Copilot, Cursor, Claude Code, Continue — under the hood, all four are similar: an editor or terminal, a context-gathering pipeline, an LLM, a diff applier. The differences are in which context gets gathered and how applies and reviews are gated.

The loop

Capture intent → gather context (open files, repo grep, embeddings) → call LLM → produce patch → apply → run → iterate.

The differences

Copilot is inline-completion. Cursor adds chat + multi-file edits. Claude Code adds full agent mode. Continue is the open-source pluggable one.

01Why AI coding works.

Code is dense, structured, repetitive at a pattern level. An LLM trained on github-scale corpus knows the patterns. Given the right context — the file, the imports, the recent edits, the project's conventions — predicting the next 5 lines is a tractable task. The model isn't reasoning about the codebase the way you do; it's producing the most-likely continuation, conditioned on context.

The product-engineering challenge isn't "make the model smarter." It's gather the right context cheaply, apply the change reliably, gate the side effects safely. Each tool is a different bundle of answers.

"The bottleneck isn't the model. It's the context the model gets to see."

02Context is everything.

The static context

Open file. File's imports. Recently-edited lines. Cursor position. This is the cheap-and-always-available context. Copilot v1 used only this. Quality of completions correlates with how much of it you can fit in the prompt without blowing past the model's window.

The repo-aware context

"Where is UserService defined? What does it do?" — for these, the tool indexes the repo. Two approaches: (1) tree-sitter / language-server symbol index for exact lookups; (2) embedding index over chunks for fuzzy semantic queries. Modern tools use both. Cursor and Claude Code can grep the codebase, read additional files, follow references.

The conversation context

"That looked wrong, try again with the X library." Chat-style tools carry a conversation history; agentic tools carry the trajectory of recent actions. The model uses recent feedback to course-correct. The longer the conversation, the more context budget is spent on history vs. new code.

Fig. AEach tool weighs these inputs differently. The bundle is what determines "good" vs "uncanny."

03Three interaction modes.

ModeWhat it doesTool
Inline completionGhost-text suggestion as you type. Accept with Tab.Copilot, Cursor Tab, Continue
Chat / inline editHighlight code, describe change, model produces a diff.Cursor, Continue, Copilot Chat
AgentDescribe a multi-step task; model plans, edits multiple files, runs commands, iterates.Claude Code, Cursor Agent, Aider, OpenHands

Each mode has a different appropriate task size. Inline = one statement to one block. Chat = a function rewrite. Agent = a feature spanning several files. Mismatching task to mode is the first reason teams find AI tools "disappointing"; pick the right grain.

04Apply, review, run.

Diffs are the unit

The model emits a patch (unified diff or structured edit). The tool validates it parses, applies it, optionally shows you the diff for review. Reject parse failures quickly — the model will sometimes hallucinate file content; the diff apply step is the canary.

Auto-run is the line you cross deliberately

Agent mode can run commands. Sandboxed by default — a non-root shell, ideally in a container. Auto-run on user machines is the same risk class as curl | sh. Most tools default to "ask before running"; the convenience of "yolo mode" is the security ceiling.

Test loop tightens quality

The model emits code → run the tests → feed the test output back. Failures become next-iteration context. This is the loop that turns 60% accuracy into 90% accuracy; the model can fix its own bugs if it can see the test result.

Review remains human

AI-generated code passes tests but can be subtly wrong — using a deprecated API, introducing a security smell, missing an edge case the tests don't cover. The reviewer is still you. Run it through CI, eyeball the diff, push.

"The AI does typing. The reviewer is still the engineer."

05Decision guide.

Most-of-the-time coding

Cursor or Copilot. Inline completion + chat. Stays in your way; gets out fast.

Full feature delegation

Claude Code or Cursor Agent. Describe what you want; supervise the loop. Best for refactors and well-spec'd tasks.

Open-source, local model

Continue + Ollama. Lower latency, no per-token cost, weaker model. Fine for tab-completion-grade tasks.

Privacy-strict environment

Self-hosted code-completion (Tabby, Codeium for Enterprise). Your code never leaves your network.

Pair-programming style

Claude Code or Aider with chat. Long-running session; the agent stays oriented on the task.

You're new to AI coding

Start with inline completion. Get comfortable. Graduate to chat. Try agent mode once you understand what context the tools have.
