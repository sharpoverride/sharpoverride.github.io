One process, a million sockets.

Apache forked a thread per connection and died at ~10k. NGINX uses an event loop per worker and lives at 1M. The architectural difference is one of the most consequential in web infrastructure history.

The trick

Non-blocking I/O + event loop (epoll/kqueue) + small number of worker processes pinned to CPU cores. No thread-per-connection.

The C10K problem

The 1999 paper that named the wall — Dan Kegel's "How to handle 10,000 simultaneous connections." NGINX (2004) was the answer.

01The thread-per-connection wall.

Apache prefork: one process per connection. ~10 MB per process. 10k connections = 100 GB RAM. Doesn't fit. Apache worker MPM: one thread per connection. ~1 MB per thread stack. 10k = 10 GB; still a lot. Plus the kernel scheduling cost of context-switching among thousands of threads.

The fundamental issue: most connections are idle most of the time. A web server is waiting on the network 99% of the wall-clock. Allocating a thread to each idle connection is paying for the cost of capacity without using it.

"The C10K problem isn't a hardware limit. It's an architectural one."

02NGINX architecture.

Master + workers

One master process (privileged, manages configuration and workers) and N worker processes (handle all connections). Typical: one worker per CPU core. 8 cores → 8 workers. Each worker is single-threaded.

Event loop per worker

Each worker runs an event loop. It calls epoll_wait() (Linux) or kqueue() (BSD) and gets a list of sockets that are ready to do something. For each ready socket: read what it can, write what it can, move on. Never blocks. A single worker can manage tens of thousands of concurrent sockets.

Why epoll/kqueue scale

The old select() and poll() were O(N) in number of file descriptors per call. epoll is O(1) — it returns only the FDs that are actually ready. You can monitor 1M FDs and the syscall to find ready ones is essentially constant time. This is the kernel feature that made the C10M era possible.

Fig. AThe architecture that broke C10K. Same shape used by Node.js, Go's runtime, Tokio, and modern Rust async.

03Tuning for 1M.

File descriptor limits

OS defaults are 1024. Bump to 1M+: /etc/security/limits.conf for the user; fs.file-max sysctl for the system. NGINX config: worker_rlimit_nofile 1000000;.

Worker connections

NGINX directive worker_connections 50000; — per worker. Total capacity ≈ workers × connections. With 8 workers × 50k = 400k connections; doubled to 1M with tuning and 16-core hardware.

Ephemeral port range

If NGINX is proxying to backends, it needs ephemeral ports for outbound connections. Default range is ~28k ports. Expand: net.ipv4.ip_local_port_range = 1024 65535. Plus keepalive to upstream so you don't burn ports per request.

TIME_WAIT and reuse

Closed sockets sit in TIME_WAIT for ~60s. At 1M conn/sec, the table fills. net.ipv4.tcp_tw_reuse = 1; tune timeout if needed.

TLS terminating is the CPU cost

1M idle HTTPS connections is the easy case. 1M actively-handshaking is CPU-bound. Session resumption tickets, OCSP stapling, modern ciphers (TLS 1.3) cut the per-connection cost. ECDSA certs are faster than RSA.

04Decision guide.

Reverse proxy / load balancer

NGINX is the default. Or HAProxy if you prefer config style. Envoy if you want L7 features and a control plane.

Static file serving

NGINX (or just a CDN). NGINX serves static files faster than almost anything; combine with sendfile() and you're at line rate.

API gateway with logic

NGINX + Lua (OpenResty) for scripted behaviour. Or Envoy if you want WASM filters and a service mesh control plane.

Massive concurrent connections (chat, push)

NGINX in front + dedicated app-tier (SignalR, Erlang, Go). NGINX terminates TLS, holds the WebSocket, forwards.

Cloud-native, K8s

NGINX ingress controller or Envoy-based (Istio, Contour). Same architectural shape; different operational story.

Trying to scale Apache

Switch to NGINX. The work is config translation, not architectural change. Most Apache configs translate to NGINX in a day.

"NGINX wins because Apache won the previous decade. The architectural idea has bled into everything else."
