The headers you'll actually use.

There are hundreds of HTTP headers. Most you'll never see. The two dozen that matter — caching, security, content type, conditional requests, rate limits — are worth learning. The others are vocabulary you can look up.

The categories

(1) Caching. (2) Content. (3) Auth. (4) CORS. (5) Security. (6) Rate-limit / metadata.

The big shift

HTTP/2 and HTTP/3 compress headers (HPACK / QPACK). The "headers are expensive" lore is mostly false in modern HTTP.

01Caching.

HeaderWhat it does
Cache-Controlpublic, max-age=3600, s-maxage=86400, immutable, stale-while-revalidate=60
ExpiresAbsolute date; superseded by Cache-Control
ETagOpaque version tag; client sends back as If-None-Match for 304
Last-ModifiedSame idea, timestamp-based; weaker
Vary"Cache by these request headers too" — careful, fragments cache
AgeHow long since the origin served this; set by CDN/proxy

Cache-Control is the most-important-and-most-skipped header in production. See CDN chapter; CDN hit rates are mostly a story of getting this right.

04CORS.

Cross-Origin Resource Sharing. Browser security mechanism; backend has to opt in to cross-origin requests. Get the headers right or every API call from a different domain fails.

HeaderSet onWhat
OriginRequestCalling page's origin (browser fills in)
Access-Control-Allow-OriginResponsehttps://example.com or * (the latter blocks cookies)
Access-Control-Allow-CredentialsResponsetrue if cookies are needed
Access-Control-Allow-MethodsPreflight responseGET, POST, PUT, DELETE
Access-Control-Allow-HeadersPreflight responseCustom headers the request will use
Access-Control-Max-AgePreflight responseHow long the browser may cache the preflight (seconds)

05Security.

HeaderWhat it does
Strict-Transport-SecurityHSTS — force HTTPS for max-age seconds (see HTTPS)
Content-Security-PolicyCSP — what scripts/styles/images are allowed; the most-misunderstood header
X-Content-Type-Options: nosniffDisable MIME-sniffing — prevents some XSS vectors
X-Frame-Options / frame-ancestorsClickjacking defence
Referrer-PolicyWhat goes in Referer on outgoing links
Permissions-PolicyRestrict camera/mic/geolocation per page

06API metadata.

HeaderWhat it does
X-RateLimit-Limit / Remaining / ResetRate-limit budget; standard since Stripe popularised it
Retry-AfterHow long to wait before retrying (response to 429 or 503)
Idempotency-KeyClient-supplied retry key for write idempotency (see idempotency)
X-Request-IDTrace-id propagated end-to-end
Sunset / DeprecationTells callers an endpoint or version is going away (see API versioning)
Server-TimingPerformance breakdown sent from server to browser dev tools
"X-" prefix is deprecated

RFC 6648 (2012) deprecated the X- convention for custom headers. New headers should use plain names. But X-RateLimit-* stuck because it was useful before standardisation; nobody renamed.

"Headers are how you do mature API design without overloading the body."
