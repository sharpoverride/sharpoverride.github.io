The security vocabulary.

Software engineers regularly read security tickets, threat models, and audit reports. The vocabulary is dense; the categories are real; the failure modes are recurrent. Twenty terms that come up most often, sorted into buckets that make them stick.

01Identity & access.

TermWhat it means
Authentication (AuthN)Who are you? Password, MFA, passkey, cert.
Authorisation (AuthZ)What are you allowed to do? Roles, scopes, policies.
SSOSingle sign-on. Sign in once at an IdP; that token authenticates you to many apps.
OAuth 2.0Delegated authorisation. "App can act on user's behalf to Service X."
OIDCOAuth 2.0 + identity layer. The standard for user login in 2026.
SAMLOlder XML-based SSO; still common in enterprise.
MFA / 2FAMulti-factor: something you have + know + are. Phishing-resistant if hardware key.
Passkey / WebAuthnPublic-key auth that bypasses passwords entirely. The post-password world.
RBAC vs ABACRole-based (admin/editor) vs attribute-based (any policy expression). RBAC fine for most; ABAC for nuanced.
Zero Trust"Never trust, always verify." No implicit trust from network position. Per-request authz.

02Crypto fundamentals.

TermWhat it means
Symmetric / asymmetricSame key for encrypt/decrypt vs key pair (public encrypts, private decrypts).
HashOne-way function: input → fixed-length digest. SHA-256, SHA-3.
HMACHash-based message authentication. Proves the message wasn't modified and came from someone who knows the secret. See webhooks.
KDFKey Derivation Function. Slow hashes for password storage: argon2id, bcrypt. See password storage.
Public-key infrastructure (PKI)Certs, CAs, chains. See HTTPS.
HSM / KMSHardware Security Module / Key Management Service. Keys never leave the secure store; signing happens inside.
JWTJSON Web Token. Signed payload carrying claims. See JWT vs sessions.

03Common attacks.

AttackWhat it isDefence
SQL injectionUser input concatenated into a SQL queryParameterised queries, always
XSS (Cross-site scripting)Attacker JS runs in another user's browserEscape output; CSP header
CSRFAuthenticated browser tricked into making a requestSameSite cookies; CSRF tokens
PhishingTricking user to authenticate to attackerPasskeys / WebAuthn (origin-bound)
Credential stuffingReplay leaked passwords on your siteMFA; passkeys; rate limit on auth
SSRF (Server-side request forgery)Server tricked into requesting internal URLsAllow-list outbound; restrict IMDS access
RCE (Remote code execution)Attacker runs code on your serverInput validation; sandboxing; least privilege
Supply-chainCompromised dependencySBOM; pin versions; signature verification
DDoSOverwhelm with trafficCDN; WAF; rate limit; auto-scale capacity headroom
Man-in-the-middleEavesdropping / tampering on the wireHTTPS / mTLS; cert pinning for clients

04Defences & frameworks.

TermWhat it is
OWASP Top 10The canonical "what to fix first" list for web apps. Refreshed every few years.
NIST CSFUS gov't risk framework. Identify, Protect, Detect, Respond, Recover.
Threat modelStructured thinking about what could go wrong. STRIDE, PASTA, etc.
Defence in depthMultiple layers. WAF + auth + IAM + audit log. No single point of failure for security.
Principle of least privilegeEvery identity has only the permissions it needs. Default deny.
Secrets managementVault, AWS Secrets Manager, Azure Key Vault. Short-lived; rotated; auditable.
SIEMSecurity Information & Event Management. Centralised log aggregation + detection.
SOC / SOC 2Security Operations Center / the audit certification.
Pen-testAuthorised attempted break-in. Quarterly to annual; remediation tracked.
Bug bountyPay researchers for finding issues. HackerOne, Bugcrowd. The cheapest external red-team.
"Security is a vocabulary first because the surface area is too large to learn by doing without one."
